GitHub: What is it?

ï GitHub is a version control platform

ï Like Dropbox, but for code

ï Stores a history of files and their changes

ï Hosts files in the cloud; easy to share finished product

ï Serves as a plaform for developer collaboration:

⇒ Twitter Bootstrap

⇒ Makes for good drama

GitHub: Why Do Developers Use It?

ï Tracks changes: see who fixed (or broke) what

ï Stores history: can easily revert and fix mistakes

ï Allows for line-by-line feedback (useful for homework)

GitHub: Some Vocab

1. Repository

2. Clone

GitHub: Repository (Repo)

ï Basic element of GitHub

ï Contains all of a project's files (all the code)

ï One or more users can contribute to a single repository

ï Repositories are either public or private

ï By EOC, you will each create your own repo!

GitHub: Clone (Copy)

ï Copies a remote repo to your machine

ï This copy is called a local repo

ï Changes to the local repo will not affect the remote

ï Multiple users can copy the same repo

GitHub: Commit

ï A snapshot of changes to a repo

ï Think of it as "saving" your local changes

ï Commits do not change the original (remote) repo

ï Contains a message describing the changes made

GitHub: Push (Send)

ï Pushes (sends) commits to a remote repository

ï Pushing does update the remote repo

ï Allows other developers to see your changes

ï Other developers can "pull" your changes

Classes and IDs

Applying Element IDs

ï IDs are attributes (options to specify element behavior)

ï Assigned to only one HTML element per page (app)

ï Identify the element for styling/scripting

ï Cannot contain spaces


	// DO NOT DO THIS
	<p id="important title">Hello, World!</p>

	// DO THIS
	<p id="important-title">Hello, World!</p>
Targeting Element IDs

ï To style the following ID:


	<p id="important-title">Hello, World!</p>

ï Use # + the IDs name:


	#important-title {
	  color: red;
	}

ï CodePen Exercise

Applying Element Classes

ï Classes are also attributes

ï Assigned to multiple HTML elements per page

ï Identifies the elements for styling/scripting

ï Like the id attribute, class cannot contain spaces


	<p class="hello">Hello, World!</p>

	<p class="hello">Hello, Mars!</p>

	<p class="hello">Hello, Jupiter!</p>
Targeting Element Classes

ï To style the following classes:


	<p class="hello">Hello, World!</p>
	<p class="hello">Hello, Mars!</p>

ï Use . + the class's name:


	.hello {
	  color: pink;
	}

ï Add HTML classes to the previous CodePen

IDs vs. Classes

ï General rule:

⇒ Use id when you are styling ONE specific element

⇒ Use a class when you are styling a GROUP of elements

ï Note:

⇒ Elements cannot have more than one ID

⇒ Elements can have multiple classes

	.some-class {
	  color: red;
	}

	#some-title {
	  color: red;
	}

No - nothing is wrong! However, is anything being repeated?

Grouping Selectors

ï Share CSS with multiple selectors

ï Helps reduce repetitive code

ï Group selectors using a comma separator:


	h2, .some-class, #some-id {
	    color: red;
	}

The Box Model

ï Every HTML element is a box

ï Each box is composed of:

⇒ content

⇒ padding

⇒ border

⇒ margin

ï Awesome box model demo

The Box Model
The Box Model: Content

Content:

⇒ The text (and other elements) within the element's tags

The Box Model: Padding

Padding:

⇒ Space between the content and border

The Box Model: Border

Border:

⇒ The border around the element

The Box Model: Margin

Margin:

⇒ Space between the border and other elements

The Box Model: Borders

ï Borders consist of three properties:

⇒ border-width: a pixel-based value for width

⇒ border-style: the style (solid, dotted, dashed, double)

⇒ border-color: the color

ï Apply all three properties via shorthand technique:


	p {
	  border: 2px solid black;
	}
The Box Model: Padding

	// Method One: targets each side individually
	p {
	  padding-top: 5px;
	  padding-bottom: 5px;
	  padding-right: 5px;
	  padding-left: 5px;
	}

	// Method Two: starts at top, moves clockwise: right, bottom, left
	p {
	  padding: 5px 5px 5px 5px;
	}

	// Method Three: (top/bottom) (left/right)
	p {
	  padding: 5px 5px;
	}

	// Method Four: targets all four sides (top, right, bottom, left)
	p {
	  padding: 5px;
	}
The Box Model: Margin

ï Margin follows the same syntax as padding:


	p {
	  margin-top: 5px;
	  margin-bottom: 5px;
	  margin-right: 5px;
	  margin-left: 5px
	}

	p {
	  margin: 5px 5px 5px 5px;
	}

	p {
	  margin: 5px 5px;
	}

	p {
	  margin: 5px;
	}
